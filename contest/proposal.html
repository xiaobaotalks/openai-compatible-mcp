痛点 · 为什么做这个

开发者想换模型时,会被 4 道关卡同时卡住

Anthropic 把 MCP 协议开源后,所有 IDE 都在抢着接,但开发者真正想用 DeepSeek 跑 Claude/Cursor/Codex 时,链路比想象中长得多。

1
配置文件 4 份,字段名各不一样
Claude Desktop 用 mcpServers,Cursor 用 mcp_servers,Codex 用 config.toml,Claude Code 用 ~/.claude.json。
每一份都得手改,字段名 / 嵌套层级 / env 写法各不相同。
2
Codex 协议与 DeepSeek 不兼容
Codex CLI 走 /v1/responses 格式(OpenAI 较新的 Responses API),DeepSeek 只支持 /v1/chat/completions。要么自己写翻译层,要么 Codex 就用不了 DeepSeek。
3
Claude Code 强制登录,跳过难
改了 ANTHROPIC_BASE_URL 之后,Claude Code 仍要求登录 Anthropic 账号。多数用户用 DeepSeek 时根本就没账号,卡在登录界面走不下去。
4
MCP 模板项目要装一堆依赖
现有 MCP server 模板都基于官方 SDK,动辄 pip install mcp pydantic fastapi uvicorn ... 十几个包,Windows 上还要配 venv、SSL、端口冲突排查。

Claude Code 跳过登录
双写 ~/.claude.json 和 ~/.claude/settings.json,设置 hasCompletedOnboarding=true,直插 ANTHROPIC_BASE_URL 跳过弹窗。

pip install 即可
向导、翻译代理、Claude 启动器、Key 写入工具全部打进包,一次拿全,Windows / macOS / Linux 通吃。
