🔌 什么是 MCP？为什么会有 4 个 IDE 要配 4 个地方？

简单点说（不装 X）：

  • MCP（Model Context Protocol） = AI 跟外部工具/数据源"对话"的协议。你可以理解成 USB 接口——你的 IDE 是个电脑，AI 模型是个 U 盘，MCP 就是那个统一的插口标准。
  • Anthropic 2024 年开源了这个协议，所有 IDE 都在抢着接。
  • 问题是：每个 IDE 接 MCP 的方式不一样，配置文件 4 份，字段名 4 种。
想象一下：你买了 1 个 U 盘，结果苹果电脑、安卓手机、Windows、Linux 每个都要配不同的驱动，每个驱动文件放不同的文件夹、字段名都不一样。你每次切换设备就要重新折腾 30 分钟。

这就是我当时面对的现实。

⭐ 5 分钟跑通 Claude Code + DeepSeek（重点来了）

下面这个组合是我自己最常用、也是帮同事们最常配的——Claude Code + DeepSeek，全程不用登录。

Step 1：装包

pip install openai-compatible-mcp

Step 2：把 DeepSeek 的 Key 写进配置

openai-compatible-mcp --api-key sk-你的DeepSeekKey

Step 3：起向导，浏览器自动开 127.0.0.1:8989

openai-compatible-mcp

打开浏览器你会看到一个暗色的控制台（对，我特意做的，不是冷冰冰的 JSON 界面）。

Step 4：勾选 Claude Code，点"写配置"

工具会自动：

  1. 写 ~/.claude.json
  2. 写 ~/.claude/settings.json
  3. 设置 hasCompletedOnboarding=true——这就是跳过登录的关键
  4. 把 ANTHROPIC_BASE_URL 指向 DeepSeek 的 Anthropic 兼容端点

Step 5：重启 Claude Code

claude

完事。没有登录弹窗。直接进 IDE 写代码，AI 后台跑的是 DeepSeek。

我自己第一次跑通的时候，差点感动哭了——被登录弹窗折腾太久，突然畅通无阻的那种爽。

🧠 几个让我得意的"小心机"

① Codex 协议不兼容？我内置了翻译代理

Codex CLI 走的是 OpenAI 较新的 /v1/responses 接口，DeepSeek 走的是 /v1/chat/completions，两个协议不兼容。我用 Flask 写了一个本地代理（127.0.0.1:7878），自动翻译请求和响应。Codex 启动时不用任何配置，AI 模型栏选 http://127.0.0.1:7878 即可。

② 不会动你已有的 MCP server

写配置走"读 → 合并 → 写"流程，只新增本包条目，你原来配的 server 全部保留。卸载时一并清掉，不留垃圾。

③ 6 家国产 LLM 一键切

默认支持 DeepSeek 千问 MinMax mimo OpenAI / 自定义。本地跑的 Ollama / vLLM / llama.cpp 走 OpenAI 兼容模式也能一行接上。
