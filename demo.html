TRAE AI 创造力大赛 初赛 Demo

openai-compatible-mcp

把任意 OpenAI 兼容 API 接入 Claude Desktop、Cursor、Claude Code、Codex 的本地翻译代理 + 配置生成器。

Demo 简介

是什么：一个可交互的 Web Demo，展示“配置翻译 + 协议翻译”两个核心能力。

面向谁：需要用 Codex / Claude Desktop / Cursor 访问 DeepSeek、OpenAI、千问、智谱、小米 mimo 等兼容 OpenAI 接口的开发者。

主要功能：

  • 配置翻译器：粘贴 Codex / Claude 配置，自动转换成代理可用格式。
  • 协议翻译器：把 /v1/responses 转换成 /v1/chat/completions，消除 Codex 0.140+ 的兼容问题。
  • 一键体验：本页即开即用，无需安装，体验核心价值。

Demo 创作思路

1
灵感来源：我自己在多个 AI 编辑器之间切换时，经常遇到“配置格式不兼容”“API 路径不一样”的问题。
2
想解决的问题：让 DeepSeek / 千问 / 智谱等 OpenAI 兼容 API 能直接在 Codex、Claude Desktop、Cursor 里用，不用重复写配置。
3
为什么做这个方向：工具链割裂是真实痛点；做“协议翻译层”比让每个客户端去适配每个厂商更可行。

体验 1：配置翻译器

选择一个样例，或直接粘贴你的配置，查看翻译后的结果。

体验 2：协议翻译器

模拟 Codex 发出的 Responses 请求，看代理如何把它翻译成 Chat Completions 请求。

体验 3：本地一键启动

如果你有一台装有 Python 3.9+ 的机器，可以自己跑起来验证：

pip install openai-compatible-mcp openai-compatible-mcp --wizard openai-compatible-mcp --proxy

Windows 也可双击 setup\proxy-launch.cmd 启动代理。
